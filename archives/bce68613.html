delete 和 truncate 的区别:

  • delete 可以删除指定行数据；truncate 清空整张表。
  • delete 删除表中所有数据后，再次插入数据时自增字段的值是删除数据前该字段最大值加 1；truncate 清空表后，再次插入数据时自增字段的值重新从 1 开始。

聚合函数

所谓聚合，就是将多行汇总成一行；所有的聚合函数均如此——输入多行，输出一行。

聚合函数具有自动滤空的功能，若某一个值为 null，那么会自动过滤该行。

只有 select 子句和 having 子句、order by 子句中能够使用聚合函数。
count()

统计行数

max()

返回指定列的最大值

min()

返回指定列的最小值

sum()

计算指定列的数值和，如果指定列类型不是数值类型则计算结果为 0

avg()

计算指定列的平均值，如果指定列类型不是数值类型则计算结果为 0

条件查询

where

使用 where 子句过滤数据。

where 子句中常用关系运算符：

运算符 说明
=、>、< 等于、大于、小于
!=、<> 不等于
>=、!< 大于等于
<=、!> 小于等于

where 子句中常用关键字：

  • in：判断某个字段的值是否在指定集合中
  • between and：判断某个字段的值是否在指定范围中
  • like：模糊查询

  • %：匹配 0 个或多个任意字符

  • _ ：匹配 1 个任意字符

  • []：匹配集合内的字符，如 [ab] 表示可以匹配 a 或 b。

    ​ 如果加上 ^ 表示否定，不匹配集合内任何字符，如 [^ab] 表示字符不能时 a 和 b。

group by

分组规定：

  • group by 子句出现在 where 子句之后，order by 子句之前；
  • 除了汇总字段外，select 语句中的每一字段都必须在 group by 子句中给出；
  • null 的行会单独分为一组；
  • 大多数 SQL 实现不支持 group by 列具有可变长度的数据类型。

group by 经常和 having 一起使用：

表的关联关系

多对一

如：员工与部门之间的关系，一个部门可以有多个员工；而一个员工只属于某一个部门。

在多对一的表关系中，应将外键建在多的一方，否则会造成数据的冗余。

多对多

如：学生与老师之间的关系，一个学生可以有多个老师，一个老师也可以有多个学生。

通常情况下，为了实现这种关系需要定义一张中间表（亦称为连接表），该表会存在两个外键分别参照老师表和学生表。

一对一

一对一的关联关系在数据库中并不常见，因为以这种方式存储的信息通常会放在同一张表中。

多表连接查询

交叉连接

交叉连接返回的结果是被连接的两个表中所有数据行的笛卡儿积。

内连接
外连接

在使用内连接查询时我们发现：返回的结果只包含符合查询条件和连接条件的数据。但是，有时还需要在返回查询结果中包含左表、右表或两个表中的所有数据，此时就需要使用外连接查询。

外连接分为左(外)连接、右(外)连接和全外连接。

左外连接

左(外)连接：返回包括左表中的所有记录和右表中符合连接条件的记录。

右外连接

右(外)连接：返回包括右表中的所有记录和左表中符合连接条件的记录。

全外连接

MySQL 不支持。

自然连接

自然连接是一种特殊的等值连接，要求进行连接的两个关系表中必须有相同的属性列（名字相同），无须添加连接条件，并且在结果中消除重复的属性列。

查询语句的执行顺序

  1. from 子句：将表文件加载到内存中
  2. where 子句：根据条件进行过滤，为符合条件的行生成一张临时表
  3. group by 子句：根据字段将临时表进行拆分，为每个字段值再生成一张临时表；此时生成的临时表包含字段 name 和 num
  4. having 子句：对第 3 步生成的临时表进行过滤，去掉不符合条件的行
  5. order by 子句：第 4 步生成的临时表进行排序
  6. limit 子句：返回限定行数
