查询

  • distinct

    同时作用于 col1 和 col2

  • limit

限制返回的行数。可以有两个参数，第一个参数为起始行，从 0 开始；第二个参数为返回的总行数。

where

用 where 子句进行数据过滤

where 子句可用的操作符：

  • = - 等于
  • < - 小于
  • > - 大于
  • <> or != - 不等于
  • <= or !> - 小于等于
  • >= or !< - 大于等于
  • between
  • is null

where 子句中还可以使用通配符，但是只能用在文本列上：

  • % - 匹配 0 个或多个任意字符
  • _ - 匹配 1 个任意字符
  • [] - 匹配集合内的字符，如 [ab] 表示可以匹配 a 或 b。如果加上 ^ 表示否定，不匹配集合内任何字符，如 [^ab] 表示字符不能时 a 和 b。
group by

分组规定：

  • group by 子句出现在 where 子句之后，order by 子句之前；
  • 除了汇总字段外，select 语句中的每一字段都必须在 group by 子句中给出；
  • null 的行会单独分为一组；
  • 大多数 SQL 实现不支持 group by 列具有可变长度的数据类型。
having

where 过滤行，having 过滤分组，行过滤先于分组过滤。

联合查询

当想要的数据在多个不同的表中，需要使用联合查询，此时需要连接多个表。

join 关键字用来连接多个表，条件语句使用 on 而不是 where。

连接可以替换子查询，并且效率一般比子查询更快。

  • 自然连接 - natural join

自然连接是一种特殊的等值连接，要求进行连接的两个关系表中必须有相同的属性列（名字相同），无须添加连接条件，并且在结果中消除重复的属性列。

  • 内连接 - inner join

内连接基本与自然连接相同，不同之处在于自然连接是同名属性列的连接，而内连接则不要求两属性列同名，可以用 using 或 on 来指定连接条件。

  • 自连接

自连接是内连接的一种特殊形式，连接自身。

  • 左外连接 - left (outer) join

左外连接是在两表进行自然连接，保留左表所有行，右表相应字段填充 null。

  • 右外连接 - left (outer) join

右外连接是在两表进行自然连接，保留右表所有行，左表相应字段填充 null。

  • 全外连接 - left (outer) join

    MySQL 不支持全外连接。

组合查询

使用 union 来组合两个查询，如果第一个查询返回 M 行，第二个查询返回 N 行，那么组合查询的结果一般为 M+N 行。

每个查询必须包含相同的列、表达式和聚集函数。

只能包含一个 order by 子句，并且必须位于语句的最后。

函数

数值计算

  • count() - 返回某列的行数

  • avg() - 返回某列的平均值

  • sum() - 返回某列值之和

  • max() - 返回某列最大值

  • min() - 返回某列最小值

文本处理

  • left()
  • right()
  • ltrim() - 去掉字符串左边空格
  • rtrim() - 去掉字符串右边空格
  • lower() - 将字符串转为小写字母
  • upper() - 将字符串转为大写字母
  • length() - 字符串长度

日期和时间

日期格式：YYYY-MM-DD

时间格式：HH:<zero-width space>MM:SS

  • date_format() - 返回一个格式化的日期或时间串

  • now() - 返回当前日期和时间

  • date() - 返回日期时间的日期部分

  • time() - 返回日期时间的时间部分

  • year() - 返回日期的年

  • month() - 返回日期的月

  • day() - 返回日期的天

  • hour() - 返回时间的小时

  • minute() - 返回时间的分钟

  • second - 返回时间的秒
