cookie和session

Posted on 2021-06-10 In 网络 Views:
Symbols count in article: 428

cookie 概述

在网路中，浏览器经常要向服务器请求数据，请求数据前要和服务器建立连接，但是 HTTP 请求是无状态的，也就是说浏览器每次在向服务器发送请求数据都要重新建立连接，频繁建立连接浪费资源。

cookie 的出现就是为了解决这个问题的，浏览器在第一次和服务器建立连接后，服务器会返回一些数据（cookie）给浏览器，让浏览器保存在本地。浏览器再次向服务器发送 HTTP 请求时，把本地缓存的 cookie 一起发给服务器，服务器通过 cookie 就可以判断出是哪个用户了。

cookie 只能存储少量数据，一般不超过 4KB。

cookie 的生命周期

如果不设置过期时间 expires 的话，cookie 的生命周期就是一次会话，浏览器的打开到关闭算一次会话。在浏览器关闭后，缓存的会话会被销毁。

session 概述

session 的作用和 cookie 类似。不过，不同于 cookie 存储于浏览器，session 是存储在服务器上的。

存储在服务器上的数据更安全，缺点是会占用服务器的资源。
